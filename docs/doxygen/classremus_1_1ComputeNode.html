Detailed Description

A ComputeNode is a machine that can run ComputeThread (s)

Everything necessary for a machine to serve in the ComputeNode role

A ComputeNode has connections to all of the MemoryNodes in the system. It also knows about all of the Segments at each node.

Configuring the ComputeNode entails reaching out and connecting to every MemoryNode, and getting all of the needed information from it. There's a difficulty though: Eventually, there will be ComputeThreads, and those ComputeThreads are going to need to have Segments for their one-sided operations. To avoid registering lots of segments, ComputeNode makes a big one, then chops it up for its ComputeThreads. This lets remus only make one registration per connection, while still supporting many threads.
